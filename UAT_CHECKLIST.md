# User Acceptance Testing (UAT) Checklist

## For Customers (20-30 participants)
1. **Registration & Login**
    - [ ] Create a new account
    - [ ] Log in with the newly created account
    - [ ] View profile information
2. **Browsing Cars**
    - [ ] Browse cars on the homepage
    - [ ] Filter cars by city/location
    - [ ] View car details
3. **Chatbot Interaction**
    - [ ] Ask the chatbot for car availability
    - [ ] Ask about booking process
    - [ ] Ask about car prices
4. **Booking Process**
    - [ ] Select a car and start the booking process
    - [ ] Fill in pickup and return details
    - [ ] Upload required documents (KTP/SIM)
    - [ ] Confirm booking
5. **Payment**
    - [ ] Proceed to payment via Midtrans
    - [ ] Complete payment using a sandbox/test method
    - [ ] Verify booking status changes to 'Disewa'
6. **History & Tracking**
    - [ ] View booking history
    - [ ] Check booking status

## For Admins (5-10 participants)
1. **Dashboard Overview**
    - [ ] View current transactions on the dashboard
    - [ ] Check total revenue and bookings
2. **Car Management**
    - [ ] Add a new car to the fleet
    - [ ] Edit existing car details (price, status, image)
    - [ ] Mark a car as unavailable for maintenance
3. **Transaction Management**
    - [ ] View and manage customer bookings
    - [ ] Update transaction status (Confirm/Reject)
    - [ ] View uploaded customer identity documents
4. **Branch Management**
    - [ ] View list of branches
    - [ ] Edit branch information
5. **Reporting**
    - [ ] View or export transaction reports

## Feedback Collection
- **Ease of Use (1-5):** How intuitive is the booking flow?
- **Chatbot Helpfulness (1-5):** Did the chatbot answer your questions correctly?
- **Payment Experience (1-5):** Was the payment process smooth?
- **Overall Satisfaction (1-5):** How would you rate the overall experience?
- **Comments/Suggestions:** What can we improve?
